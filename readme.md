### 3. Описание реализации.
- Главный поток генерирует набор исходных данных и добавляет их в потокобезопасную очередь. Дальше создаются независимые потоки, работающие параллельно. В  программе реализован класс TaskQueue. В нем написаны методы добавления и извлечения задач, синхронизации  данных. Также в  программе реализована функция void worker(unsigned int id, TaskQueue& q). Она имитирует обработку задачи и выполняет потокобезопасный вывод в консоль. В main() определяется количество потоков процессора при помощи hardware_concurrency(), но если оно вернуло 0, то программа определяет количество потоков процессора как 1. Для Windows вывод в консоли настроен в кодировке UTF-8.
- Потокобезопасная очередь реализована с помощью std::mutex. std::queue не обладает потокобезопасностью, что приводит к гонке данных, если происходит одновременный доступ нескольких потоков без сихронизации данных. В методе push(const int task) класса TaskQueue поток захватывает mutex через std::lock_guard, добавляет задачу и пробуждает один из ожидающих потоков при помощи cv.notify_one(). В методе pop(int& task) также поток захватывает mutex через std::unique_lock. std::condition_variable требует unique_lock, так как нам необходимо атомарно освободить mutex и перевести поток в состоянии ожидания при вызове wait(). В методе wait() мы передаем предикат в лямбде-выражении, что защитит нас от ложных пробуждений и сработает только в случае, когда у нас есть задачи или установлен флаг stop, а дальше идет проверка очереди, которая позволяет обработать корректно. В методе stopQueue() главный поток захватывает мьютекс, устанавливает флаг stop = true и вызывает cv.notify_all(), пробуждая все рабочие потоки для корректного завершения. Мьютекс здесь необходим чтобы изменить общее состояние stop. Предикат в методе pop(int& task) в wait() проверяет условие !tasks.empty() || stop, которое не дает потоку обратиться к пустой очереди и зависнуть.
- std::condition_variable применяется, чтобы не использовать busy-wait, которое запрещено по условиям. std::condition_variable помогает нам не загружать потоки, которые не задействуются при выполнении программы. Применяется она в методе push(const int task) notify_one(), чтобы пробудить один из спящих потоков, в методе stopQueue() notify_all пробуждает все ожидающие потоки. В методе pop(int& task) мы используем wait() и предикат в виде лямбда-выражения, чтобы не допустить ложных срабатываний и не обращаться к пустой очереди.
- Поток завершается в функции worker(). В классе TaskQueue в методе stopQueue() поток захватывает мьютекс и устанавливает флаг stop = true. Через std::condition_variable уведомляются все потоки, чтобы завершить спящие потоки. Далее потоки продолжают работу в методе pop(int& task), который возвращает true, если есть задачи, а если нет задач и установлен флаг stop = true, то возвращает false. Как только метод вернет false, тогда это приводит к выходу из цикла worker. После выхода рабочих потоков из цикла главный поток корректно завершает их с помощью join().
